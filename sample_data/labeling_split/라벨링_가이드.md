# K-pop 거래 게시글 라벨링 가이드

## 1. 컬럼 설명

| 컬럼명 | 설명 | 예시 |
|--------|------|------|
| post_id | Reddit 게시글 고유 ID | 1oz8j3h |
| title | 게시글 제목 | [WTS][USA] Twice Photocards |
| author | 작성자 닉네임 | kpop_seller123 |
| author_flair | 작성자 거래 이력 (중요!) | Trades: 52 |
| transaction_type | 거래 유형 | WTS(판매), WTB(구매), WTT(교환), ISO(구함) |
| country | 거래 가능 국가 | USA, WW(전세계), UK 등 |
| flair | 게시글 카테고리 | Photocard, Album, Ticket 등 |
| score | 추천수 (upvote - downvote) | 5 |
| comment_count | 댓글 수 | 3 |
| selftext | 게시글 본문 내용 | (상세 설명) |
| created_timestamp | 게시글 작성 시간 | 2024-12-10T15:30:00 |
| permalink | Reddit 게시글 링크 | /r/kpopforsale/comments/... |
| first_image_url | 첫 번째 이미지 URL | https://... |
| is_gallery | 여러 이미지 여부 | TRUE/FALSE |
| is_scam | **라벨링 대상** | TRUE/FALSE/(비워두기) |
| label_notes | 판단 근거 메모 | (자유 기재) |
| labeled_by | 라벨링 담당자 | 시영님 |
| labeled_at | 라벨링 일시 | 2024-12-16 |

## 2. 라벨링 기준

### 2.1 정량적 판단 기준표

| 항목 | TRUE (사기 의심) | FALSE (정상) |
|------|------------------|--------------|
| 거래 횟수 (author_flair) | 0~2회 또는 없음 | 10회 이상 |
| 이미지 | 없음 | 있음 (is_gallery=TRUE 또는 first_image_url 존재) |
| score (추천수) | 0 이하 (downvote) | 3 이상 |
| 댓글 수 | 0개 | 1개 이상 |
| 가격 정보 | 불명확하거나 비정상 | 명확한 $ 가격 명시 |
| 결제 수단 | 언급 없음 또는 외부 유도 | PayPal G&S 명시 |

### 2.2 의심 키워드 (TRUE 방향)
- 급처 표현: `urgent`, `asap`, `need gone`, `must go`, `desperate`, `quick sale`
- 외부 연락 유도: `DM me`, `WhatsApp`, `Telegram`, `KakaoTalk`, `Line`
- 선입금 유도: `payment first`, `pay before`

### 2.3 신뢰 키워드 (FALSE 방향)
- 안전 결제: `PayPal G&S`, `PayPal Goods & Services`, `G&S only`
- 인증: `timestamp`, `proof`, `verification`
- 배송 정보: `shipped`, `tracking`, `USPS`, `stamped`

## 3. TRUE (사기 의심) 예시

### 예시 1: 거래 이력 없음 + 이미지 없음 + 반응 없음
```
post_id: 1otjchw
title: [WTS][India] Selling Dynamite cassette, BE photobook and PCs
author_flair: (없음) → 거래 이력 0회
score: 0, comments: 0
이미지: 없음
본문: (없음)

판단 근거:
- 거래 이력이 전혀 없음
- 이미지 증빙 없음
- 본문 설명 없음
- 커뮤니티 반응(추천/댓글) 없음
```

### 예시 2: 급처 표현 + 신규 계정
```

score: 0, comments: 0
본문: "Have over 500 cards i desperately need rid of..."

판단 근거:
- "desperately" 급처 표현 사용
- 거래 이력 없음
- 미스터리 팩 (내용물 불명확)
- 커뮤니티 반응 없음
```

### 예시 3: ASAP 강조 + 신규 계정
```

- 대면 거래 유도 가능성
```

## 4. FALSE (정상) 예시

### 예시 1: 다수 거래 이력 + 상세 정보
```

title: [WTS][USA] Various POBs and Photocards ZB1, Red Velvet, Riize, NCT
author_flair: Trades: 52 → 52회 성공적 거래
score: 3, comments: 2
이미지: 있음 (갤러리)
본문: "Prices don't include shipping yet: $1 stamped USA, $5 tracked USA..."

판단 근거:
- 52회 거래 이력 (높은 신뢰도)
- 가격 명확히 명시 ($1, $5)
- 배송 방법 상세 (stamped, tracked)
- 이미지 증빙 있음
- 긍정적 커뮤니티 반응
```

## 5. 애매한 경우 처리

판단이 어려운 경우:
1. `is_scam` 컬럼을 **비워두고**
2. `label_notes`에 애매한 이유를 작성

예시:
```
is_scam: (비워둠)
label_notes: "거래이력 0회이나 PayPal G&S 명시, 이미지 있음. 신규 판매자일 가능성"
```

## 6. 라벨링 작업 방법

### 6.1 기본 절차
1. 구글 시트에서 배정된 데이터 확인
2. 원본 게시글과 프로필 직접 확인 (아래 참고)
3. `is_scam` 컬럼에 TRUE/FALSE 입력 (그래도 애매하면 비워두기)
4. `label_notes`에 판단 근거 간단히 작성
5. `labeled_by`에 본인 이름, `labeled_at`에 오늘 날짜 입력

### 6.2 원본 게시글 확인 방법
`permalink` 컬럼의 URL을 클릭하거나 복사해서 브라우저에서 열기

원본에서 확인할 것:
- 실제 이미지 (시트에 안 보이는 추가 이미지들)
- 댓글 내용 (다른 유저들 반응, 경고 댓글 등)
- 삭제 여부 ([deleted] 표시)

### 6.3 판매자 프로필 확인 방법
`https://www.reddit.com/user/작성자닉네임` 접속

예시:
- author가 `kpop_seller123`이면
- `https://www.reddit.com/user/kpop_seller123` 접속

프로필에서 확인할 것:
- 계정 생성일 (너무 최근이면 의심)
- 다른 거래 게시글 히스토리
- 카르마 점수
- 다른 서브레딧 활동 내역

### 6.4 판단 우선순위
1. 시트 데이터만으로 명확하면 → 바로 라벨링
2. 애매하면 → 원본 게시글 확인
3. 그래도 애매하면 → 프로필 확인
4. 최종적으로 애매하면 → is_scam 비워두고 label_notes에 이유 작성
